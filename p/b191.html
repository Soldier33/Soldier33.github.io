  1. Oracle数据库服务器工作模式

    主要有两种：专用服务器操作模式和共享服务器模式。

    专用服务器模式是指Oracle为每个用户进程启动一个专门的服务器进程

    线程共享服务器模式是指一个服务器进程可以被多个用户进程共享（调度进程的帮助下）

  2. Oracle应用结构

    1、多数据库的独立宿主结构

    在物理上只有一台服务器，服务器上有一个或多个硬盘。但是在功能上是多个逻辑数据库服务器和多个数据库

​ 2、客户/服务器结构

​ 数据库服务器的管理和应用分布在两台计算机上，客户机上安装应用程序和连接工具，通过Oracle专用的网 络协议SQL *Net建立和服务器的连接，发出数据请求。服务器上运行数据库，通过网络协议接收连接请求， 将执行结果回送给客户机。

​ 3、分布式结构

​ 在这种结构中，分布式数据库系统在逻辑上是一个整体，但在物理上分布在不同的计算机网络里，通过连接 网络连接在一起。

  1. Oracle基本概念

    表空间（Tablespace）：是数据库的逻辑划分，每个数据库至少有一个表空间（system表空间）。

    数据文件（DataFile）：每个表空间由同一磁盘上的一个或多个文件组成，这些文件叫数据文件。建立新表空间需要建立新的数据文件。

    实例（Instance）：也称为服务器（server），是存取和控制数据库的软件机制，它由系统全局区SGA和后台进程组成。

    常用的数据库类型有事务处理类、数据仓库类。

    两种数据库类型的区别在于：事务处理类一般数据量小，DML频繁，而数据仓库类数据量大，DML少；且两种数据库类型适用与不同类型的应用。

    事务处理类型：主要针对具有大量并发用户连接，并且用户主要执行 简单事务处理的应用常用数据库环境。事务处理数据库的典型应用有 银行系统数据库、Internet电子商务数据库、证券交易系统数据库等。

    数据仓库类型：主要针对有大量的对某个主题进行复杂查询的应用环境。数据仓库的典型应用有客户订单研究、支持呼叫、销售预测、采购模式以及其他战略性业务问题的历史数据研究。

  1. 必须启动的两个服务

    OracleServiceORCL：数据库服务（数据库实例），是Oracle核心服务，是数据库启动的基础，只有该服务启动，Oracle数据库才能正常启动。（必须启动）
    OracleOraDb11g_home1TNSListener：监听器服务，该服务只有在远程访问数据库时才需要（无论远程计算机还是本地计算机，凡是通过Oracle Net网络协议连接数据库都属于远程访问）。（必须启动）

    OracleDBConsoleorcl：Oracle控制台服务，即企业管理器服务。只有该服务启动了，才可以使用Web方式的企业管理器管理数据库。（非必须启动）

  4. oracle数据库体系结构的构成

    存储结构：物理存储结构和逻辑存储结构

    软件结构：内存结构和后台进程结构

    物理存储结构的组成：

    1.数据文件

    2.控制文件

    3.重做日志文件

    4.归档重做日志文件

​ 5.初始化参数文件

​ 6.跟踪文件

​ 7.告警文件 

 **逻辑存储结构的组成**：结构由数据块、区、段、表空间和数据库构成

​ 表空间是Oracle数据库最大的逻辑存储单元

​ 表空间分类：系统表空间 ，非系统表空间，撤销表空间，临时表空间，用户表空间

​ oracle数据库内存结构的组成：系统全局区（SGA）和程序全局区（PGA）。

​ （1）数据库实例由各种高速缓冲池以及后台进程组成。

​ （2）SGA构成：数据高速缓冲区，共享池，重做日志缓冲区，大型池，Java池，流池，

​ 其它结构（如固定SGA、锁管理等）

​ （3）数据高速缓冲区：脏缓存块，空闲缓存块，命中缓存块，干净缓存块

​ （4）PGA构成：排序区，游标信息区，会话信息区，堆栈区

​ 后台进程的组成：

​ DBWR 数据库写入进程
​ LGWR 日志写入进程
​ CKPT 检查点进程
​ SMON 系统监控进程
​ PMON 进程监控进程
​ ARCH 归档进程
​ RECO 恢复进程
​ LCKn 锁进程
​ Dnnn 度进程等

（1）简述oracle数据库后台进程DBWR何时启动。

​ 答：

​ 1.在执行INSERT、UPDATE等操作时 ，没有足够的空闲块时。

​ 2.当检查点发生时 。

​ 3.当数据缓存的LRU列表的长度达到初始化 DB_BLOCK_WRITE_BATCH指定值的一半时 。

​ 4.发生超时的时候 。

（2）简述oracle数据库后台进程LGWR何时启动

​ 答：

​ 1.当用户通过COMMIT语句提交当前事务时。

​ 2.当重做日志缓存被写满三分之一时。

​ 3.当DBWR进程开始将脏缓存块写入数据文件时。

​ \4. 当发生一次超时，将启动LGWR 。

  3. 模式是指一系列逻辑数据结构或对象的集合。（如SYS模式）

    一个模式只能被一个数据库用户所拥有

    同一模式中数据库对象的名称必须惟一，而在不同模式中的数据库对象可以同名

  4. Oracle ** **数据库中创建表的方法有哪几种。

    sql语句：

    一、直接创建：create table tablename；

    二、 ** **创建与表A结构相同的空表：

    create table a_tmp as select * from A where 1=0；（即为假）

    三、 ** **复制表A全部（或部分）数据：

    create table a_tmp as select * from A (where a.xxx=xxx)；

  5. 约束作用
    保证数据的完整性，是在表中定义的用于维护数据库完整性的一些规则。通过对表中列定义约束，可以防止在执行DML操作时，将不符合要求的数据插入到表中。

  6. 约束类型
    PRIMARY KEY
    UNIQUE
    CHECK
    FOREIGN KEY
    NULL/NOT NULL

  7. 索引的作用

    索引是为了加速对表中元组的检索而创建的一种分散存储结构；
    是对表而建立的，由除存放表的数据页面以外的索引页面组成，独立于被索引的表；
    通过使用索引加速行的检索，但减慢更新的速度；
    快速定位数据，减少磁盘 I/O；
    Oracle自动使用、维护索引

  8. 索引类型：

    B-树索引
    位图索引
    函数索引
    唯一性索引与非唯一性索引
    单列索引与复合索引

    在emp表的last_name列上创建一个非唯一性索引。
    CREATE INDEX emp_lname_indx ON emp (last_name) ;
    在emp表的email列上创建一个唯一性索引。
    CREATE UNIQUE INDEX emp_email_indx ON emp(email);
    在emp表的job_id列上创建一个位图索引。
    CREATE BITMAP INDEX emp_job_indx ON emp(job_id) ;
    基于emp表的first_name列创建一个函数索引。
    CREATE INDEX emp_fname_indx ON emp (UPPER(first_name)) ;

  9. 索引使用原则

    导入数据后再创建索引
    在适当的表和字段上创建索引
    合理设置复合索引中列的顺序
    限制表中索引的数目
    选择存储索引的表空间

  10. 视图是从一个或多个表或视图中提取出来的数据的一种逻辑表现形式。

    提供一定的数据范围成为视图给用户

    视图是一个虚拟表，视图并不在数据库中存储数据值，数据库中只在数据字典中存储对视图的定义。

**视图分为简单视图和复杂视图两类。**

  1. 视图作用：

    1.可以限制对基表数据的访问，只允许用户通过视图看到表中的部分数据。

    2.可以使复杂的查询简单化。

    3.提供了数据的透明性，用户并不知道数据来自于何处。

    4.提供了对相同数据的不同显示。

  2. 简述序列的作用及使用方法。

    序列: 是oacle提供的用于产生一系列唯一数字的数据库对象。

    作用：

    1．自动提供唯一的数值（可以为表中的记录自动产生唯一序号）

    2．共享对象（由用户创建并且可以被多个用户共享）

    3．主要用于提供主键值（典型应用是生成主键值，用于标识记录的唯一性）

    4. 将序列值装入内存可以提高访问效率（使用缓存可以加速序列的访问速度）

    5. 允许同时生成多个序列号（每一个序列号是唯一的）

    使用CREATE SEQUENCE语句创建序列

    START WITH：设置序列初始值，默认值为1。

    INCREMENT BY：设置相邻两个元素之间的差值，即步长，默认值为1。

    MAXVALUE：设置序列最大值。

    NOMAXVALUE：默认情况下，递增序列的最大值为1028-1，递减序列的最大值为-1。

    MINVALUE：设置序列最小值。

    NOMINVALUE：默认情况下，递增序列的最小值为1， 递减序列的最小值为-(1027 -1)。CYCLE：当序列达到其最大值或最小值后，开始新的循环。

    NOCYCLE：当序列达到其最大值或最小值后，序列不再生成值。默认选项。

    CACHE：设置Oracle服务器预先分配并保留在内存中的序列值的个数，默认为20。

    NOCACHE：不缓存序列值

  3. 对巨型表进行分区具有下列优点：
    提高数据的安全性，一个分区的损坏不影响其他分区中数据的正常使用。
    将表的各个分区存储在不同磁盘上，提高数据的并行操作能力。
    简化数据的管理，可以将某些分区设置为不可用状态，某些分区设置为可用状态，某些分区设置为只读状态，某些分区设置为读写状态。
    操作的透明性，对表进行分区并不影响对数据进行操作的SQL语句。
